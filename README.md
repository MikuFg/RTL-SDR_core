# Radar AI Detection System

Система распознавания объектов на радаре с использованием нейросетей и обработки реальных радиосигналов.

## Обзор проекта

Проект представляет собой комплексную систему для:
- Захвата и обработки радиосигналов
- Преобразования сигналов в изображения для нейросети
- Классификации объектов (самолет, птица, дрон, вертолет)
- Формирования JSON отчетов с результатами
- Визуализации в реальном времени

## Архитектура

### Основные модули

1. **`signal_processor.py`** - обработка радиосигналов
   - Захват сигналов с микрофона/радиоприемника
   - Спектральный анализ и фильтрация
   - Преобразование в изображения для нейросети

2. **`model_engine.py`** - ядро нейросети
   - Классификатор на основе ResNet18
   - Распознавание 4 классов объектов
   - Оценка уверенности и рисков

3. **`json_formatter.py`** - формирование отчетов
   - Создание структурированных JSON результатов
   - Генерация отчетов и статистики
   - Экспорт в CSV

4. **`ui_bridge.py`** - основной интерфейс
   - Симуляция радарной системы
   - Режимы работы (тестовый, реальный, пакетный)
   - Интеграция всех компонентов

## Установка

1. Клонируйте репозиторий
2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Убедитесь что модель `radar_model_v2.pth` находится в директории проекта

## Использование

### Запуск системы

```bash
python ui_bridge.py
```

### Режимы работы

1. **Симуляция с тестовыми данными** - использует изображения из `data_v2/val`
2. **Симуляция с реальными сигналами** - захватывает сигналы в реальном времени
3. **Режим реального времени** - обработка отдельных сигналов
4. **Пакетная обработка** - обработка файлов из директории


### Структура данных JSON

Каждое обнаружение генерирует JSON файл с подробной информацией об объекте:
=======
### Структура данных

```json
{
  "detection_id": "DET_20231201_143022",
  "timestamp": "2023-12-01T14:30:22",
  "object_info": {
    "type": "drone",
    "confidence": 0.85,
    "is_reliable": true
  },
  "location": {
    "azimuth": 45.5,
    "range": 5000,
    "altitude": 1500
  },
  "signal_analysis": {
    "rms": 0.15,
    "crest_factor": 8.2,
    "dominant_frequency": 1500.0
  },
  "risk_assessment": "high",
  "signal_quality": "excellent"
}
```

#### 🎯 **Описание атрибутов JSON:**

**Основные поля обнаружения:**
- **`detection_id`** - Уникальный идентификатор обнаружения (формат: `DET_YYYYMMDD_HHMMSS`)
- **`timestamp`** - Время обнаружения в формате ISO 8601

**Информация об объекте (`object_info`):**
- **`type`** - Тип объекта: `airplane`, `bird`, `drone`, `helicopter`
- **`confidence`** - Уверенность классификации (0.0 - 1.0, где 1.0 = 100% уверенность)
- **`is_reliable`** - Флаг надежности обнаружения (`true`/`false`)

**Положение объекта (`location`):**
- **`azimuth`** - Азимут (угол от 0° до 360°, направление по горизонтали)
- **`range`** - Дальность до объекта в метрах
- **`altitude`** - Высота объекта над землей в метрах

**Анализ сигнала (`signal_analysis`):**
- **`rms`** - Среднеквадратичное значение сигнала (показатель мощности, >0.1 = отлично)
- **`crest_factor`** - Пик-фактор сигнала (<20 = хорошее качество)
- **`dominant_frequency`** - Доминирующая частота в Гц (помогает идентифицировать объект)

**Оценка рисков:**
- **`risk_assessment`** - Уровень угрозы:
  - `high` - дрон (высокая угроза)
  - `medium` - вертолет (средняя угроза)
  - `low` - самолет (низкая угроза)
  - `very_low` - птица (минимальная угроза)

**Качество сигнала:**
- **`signal_quality`** - Оценка качества:
  - `excellent` - RMS > 0.1, пик-фактор < 20
  - `good` - RMS > 0.05, пик-фактор < 30
  - `fair` - RMS > 0.01
  - `poor` - RMS ≤ 0.01

## Классы объектов

- `airplane` - самолет
- `bird` - птица  
- `drone` - дрон
- `helicopter` - вертолет

## Оценка рисков

- `high` - дрон (высокий риск)
- `medium` - вертолет (средний риск)
- `low` - самолет (низкий риск)
- `very_low` - птица (очень низкий риск)

## Качество сигнала

- `excellent` - RMS > 0.1, пик-фактор < 20
- `good` - RMS > 0.05, пик-фактор < 30
- `fair` - RMS > 0.01
- `poor` - RMS ≤ 0.01

## Файлы результатов

Результаты сохраняются в директории `results/`:
- `radar_detection_YYYYMMDD_HHMMSS.json` - отдельные обнаружения
- `SESSION_YYYYMMDD_HHMMSS.json` - сессии обнаружений
- `daily_report_YYYY-MM-DD.json` - дневные отчеты

## Интеграция с Go-сервером

Система поддерживает два способа отправки данных на Go сервис:

### 🔄 **WebSocket соединение (основной метод)**
- **URL:** `ws://localhost:8080/ws`
- **Преимущества:** Реальное время, низкая задержка, постоянное соединение
- **Формат данных:** JSON объекты обнаружения

### 📡 **HTTP POST (резервный метод)**
- **URL:** `http://localhost:8080/update`
- **Используется:** Если WebSocket недоступен
- **Формат данных:**
```json
{
  "type": "real_time_update",
  "timestamp": "2023-12-01T14:30:22",
  "target_id": "TGT-101",
  "prediction": "drone",
  "confidence": 0.85,
  "location": {"azimuth": 45.5, "range": 5000, "altitude": 1500},
  "risk_level": "high",
  "signal_quality": "excellent"
}
```

### 🧪 **Тестирование без Go-сервера**

Для тестирования генерации JSON файлов:
```bash
# Запуск тестовой симуляции
python test_simulation.py
```

Эта команда создаст тестовые JSON файлы в директории `results/` без необходимости запуска Go-сервера.

## Требования к оборудованию

- Python 3.7+

- Минимум 4GB RAM
- CUDA (опционально для ускорения)

## Разработка

### Добавление новых классов

1. Обучите модель на новых данных
2. Обновите список классов в `model_engine.py`
3. Адаптируйте оценку рисков в `RadarSignalClassifier`

### Настройка обработки сигналов

Измените параметры в `RadarSignalProcessor`:
- `sample_rate` - частота дискретизации
- `fft_size` - размер FFT
- Фильтры и окна обработки
