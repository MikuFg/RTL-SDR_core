# Тесты для Radar AI Detection System

Этот каталог содержит полный набор тестов для проверки работоспособности системы распознавания объектов на радаре.

## Структура тестов

```
tests/
├── __init__.py                 # Инициализация пакета тестов
├── README.md                   # Этот файл
├── run_tests.py               # Основной тестовый runner
├── test_data_generator.py      # Генератор тестовых данных
├── test_signal_processor.py   # Тесты для обработки сигналов
├── test_model_engine.py        # Тесты для нейросети
├── test_json_formatter.py      # Тесты для форматирования JSON
├── test_ui_bridge.py          # Интеграционные тесты
└── test_data/                 # Тестовые данные (генерируются автоматически)
    ├── signals/               # Тестовые радиосигналы
    ├── images/                # Тестовые изображения
    ├── json/                  # Тестовые JSON файлы
    └── mock_model.pth         # Мок-модель для тестов
```

## Описание тестов

### 1. test_signal_processor.py

Тестирует модуль обработки радиосигналов:

- **Тестирование инициализации** `RadarSignalProcessor`
- **Генерация тестовых сигналов** с известными параметрами
- **Обработка сигналов** (фильтрация, FFT, спектрограмма)
- **Преобразование в изображения** для нейросети
- **Анализ характеристик** сигнала (RMS, пик-фактор, доминирующая частота)
- **Сохранение и загрузка** сигналов
- **Интеграционные тесты** полного конвейера обработки

### 2. test_model_engine.py

Тестирует нейросетевой модуль:

- **Инициализация** `RadarAI` и `RadarSignalClassifier`
- **Предсказания** из изображений и файлов
- **Метаданные** результатов (временные метки, информация о модели)
- **Оценка рисков** на основе типа объекта
- **Оценка качества** сигнала
- **Пороги достоверности** предсказаний
- **Интеграционные тесты** с разными размерами изображений

### 3. test_json_formatter.py

Тестирует модуль форматирования результатов:

- **Форматирование** результатов обнаружения
- **Пакетные результаты** и сводки
- **Сохранение и загрузка** JSON файлов
- **Обновления в реальном времени**
- **Отчеты об ошибках**
- **Экспорт в CSV**
- **Очистка старых файлов**
- **Генерация дневных отчетов**

### 4. test_ui_bridge.py

Тестирует основной интерфейс системы:

- **Класс `RadarTarget`** - поведение целей на радаре
- **Обновление позиций** и граничные условия
- **Захват сигналов** (успешный и неуспешный)
- **Симуляция** с тестовыми данными
- **Симуляция** с реальными сигналами
- **Режим реального времени**
- **Пакетная обработка** файлов
- **Обработка ошибок** (подключение к серверу, захват сигнала)

## Расширение тестов

### Добавление нового теста

1. Создайте новый тестовый класс в соответствующем файле
2. Наследуйте его от `unittest.TestCase`
3. Определите методы с префиксом `test_`
4. Используйте `self.assert*` методы для проверок

```python
class TestNewFeature(unittest.TestCase):
    def test_new_functionality(self):
        # Подготовка
        test_data = create_test_data()
        
        # Действие
        result = process_data(test_data)
        
        # Проверка
        self.assertEqual(result['status'], 'success')
        self.assertGreater(result['confidence'], 0.5)
```

### Мокирование внешних зависимостей

Используйте `unittest.mock` для изоляции тестов:

```python
from unittest.mock import Mock, patch

class TestWithMocks(unittest.TestCase):
    @patch('module.external_service')
    def test_with_mock(self, mock_service):
        mock_service.return_value = {'status': 'ok'}
        result = function_using_service()
        self.assertEqual(result['status'], 'ok')
```

## Устранение неполадок

### Частые проблемы

1. **Отсутствующие зависимости**
   ```
   ❌ Отсутствуют следующие модули: torch, scipy
   💡 Установите их с помощью: pip install -r requirements.txt
   ```

2. **Проблемы с захватом звука**
   ```
   ⚠️ Устройство захвата недоступно: [Errno -9999] No input device
   ```
   - Тесты продолжаются с мок-данными

3. **Проблемы с правами доступа**
   ```
   PermissionError: [Errno 13] Permission denied: 'test_file.json'
   ```
   - Убедитесь что есть права на запись в тестовую директорию

4. **Проблемы с памятью**
   ```
   MemoryError: Unable to allocate array
   ```
   - Уменьшите размер тестовых данных или увеличьте доступную память

### Полезные флаги

```bash
# Тихий режим (только итоги)
python tests/run_tests.py -v 0

# Только ошибки
python tests/run_tests.py -v 1

# Подробный вывод с трассировкой
python tests/run_tests.py -v 2
```
